Rules:

- `risk_pattern_checks` merges the default AD-0005 table with optional

- `invariants` are copied verbatim from
  `.mpl/mpl/phase0/design-intent.yaml` by `applies_to_phases`.
- `mvp` is present only when `.mpl/goal-contract.yaml` has `mvp_scope`; its
  `phases[]` is the deterministic id-set intersection of `mvp_scope` AC/AX ids
  with each phase's `goal_trace`.
