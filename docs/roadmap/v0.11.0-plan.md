# MPL v0.11.0 Implementation Plan

> **테마**: v2 Phase 2 — 구조 전환 (Gate + Hat + Agent 통합)
> **현재 버전**: v0.10.2
> **의존성**: v0.10.0 완료 (L0/L1/Sentinel/Seed 기반 구축 완료)
> **Breaking**: Gate 구조 변경, 에이전트 9개 제거, Maturity/Tier 제거

---

## 1. 목표

v2 합의안(§3-§8)의 구조적 전환을 실행한다. v0.10.0에서 구축한 기계적 강제 기반(L0/L1/Sentinel/Seed) 위에 Gate 재구조화, Hat+Floor 모델, 에이전트 통합을 적용하여 MPL의 골격을 v2로 전환한다.

### 정량적 목표

| 메트릭 | v0.10.2 (현재) | v0.11.0 (목표) |
|--------|--------------|---------------|
| 에이전트 수 | 16 | 8 |
| Gate 수 | 6 | 3 Hard + 1 Advisory |
| Phase Runner 프롬프트 | ~690줄 | ~250줄 |
| Decomposer 프롬프트 | ~650줄 | ~200줄 |
| Retry 구조 | 곱셈적 (TODO×micro-fix×phase×redecomp) | Phase-level only (tier별 차등) |

## 2. 작업 항목 (13개)

### 2.1 Gate 재구조화

| ID | 작업 | 파일 | 상세 |
|-----|------|------|------|
| **GATE-01** | 6 Gate → 3H+1A | `commands/mpl-run-execute-gates.md`, `hooks/mpl-validate-output.mjs` | Hard 1(Build+Lint+Type), Hard 2(Test+Regression), Hard 3(L1/L2 Contract Diff), Advisory(Review+PP) |

**변경 상세**:
```
현재 Gate 구조:
  Gate 0.5: lint+type         → Hard 1로 통합
  Gate 0.7: cross-boundary    → Hard 3으로 흡수 (L1/L2가 이미 대체)
  Gate 1:   tests             → Hard 2로 통합
  Gate 1.5: coverage          → 제거 (Advisory에 흡수)
  Gate 1.7: browser QA        → 제거 (non-blocking, optional)
  Gate 2:   code review       → Advisory로 통합
  Gate 3:   PP compliance     → Advisory로 통합

새 Gate 구조:
  Hard 1:   Build + Lint + Type          ($0, 기계적, binary)
  Hard 2:   Full Test Suite + Regression ($0, 기계적, binary)
  Hard 3:   L1/L2 Contract Diff Guard    ($0, 기계적, binary)
  Advisory: Code Review + PP Compliance  (LLM, non-blocking)
```

### 2.2 Hat + Floor 모델

| ID | 작업 | 파일 | 상세 |
|-----|------|------|------|
| **HAT-01** | Hat 모델 구현 | `commands/mpl-run-execute.md`, `commands/mpl-run-decompose.md` | Maturity×Tier → PP-proximity 단일 축. `depth = max(PP_proximity, floor_minimum)` |
| **HAT-02** | PP 분류 규칙 | `commands/mpl-run-decompose.md` | PP-core/PP-adjacent/Non-PP 자동 분류 heuristic + 사용자 override |
| **HAT-03** | Floor 정의 | `commands/mpl-run-execute-gates.md` | 모든 Phase 공통: build+lint+type+regression+L1 Contract |

**PP-proximity 깊이 결정**:
```
PP-core (직접 영향):     3 Hard Gate + Advisory + MCP Judge 사용 가능 + retry 3회
PP-adjacent (간접 영향): 3 Hard Gate + Advisory + retry 2회
Non-PP (무관):           Floor만 (3 Hard Gate) + retry 1회
보안/데이터:             PP 무관하게 PP-adjacent 이상으로 격상
```

**제거 대상**: `maturity_mode` (explore/standard/strict), `pipeline_tier` (frugal/standard/frontier), Cluster Ralph

## 6. 롤백 계획

v0.11.0은 breaking change가 많으므로 롤백 경로를 명시:

```
롤백 트리거:
  - exp4에서 CB 버그가 3건 이상 (v0.10.2 대비 악화)
  - 총 토큰이 100K 초과 (v0.10.2 대비 2x 이상)
  - Gate 통과 불가 (빌드/테스트 기본 실패)

롤백 방법:
  git revert --no-commit HEAD~N (Sprint별 revert 가능)
  에이전트 파일은 git history에서 복원
```

## 7. 리스크

| 리스크 | 심각도 | 대응 |
|--------|--------|------|
| Phase Runner 250줄이 불충분 | Medium | Seed 품질에 의존 — Seed가 명확하면 Runner는 단순 실행만 필요. exp4에서 검증 |
| Decomposer 200줄로 분해 품질 저하 | Medium | 불필요한 로직 제거일 뿐, 핵심 추론은 유지. opus 모델의 추론 능력에 의존 |
| Advisory Gate가 무시됨 | Low | Advisory는 참고용. Hard Gate 3개가 핵심 품질 보장 |
| PP 분류 heuristic 부정확 | Medium | 사용자 override 제공. 초기에는 보수적(PP-adjacent 기본) |
| 9개 에이전트 동시 제거 충격 | High | Sprint 2에서 한번에 제거하되, 각 에이전트의 핵심 로직이 흡수처에 이전되었는지 체크리스트로 확인 |
