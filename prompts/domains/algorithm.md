# Domain: Algorithm (복잡 로직)

## 핵심 원칙
- 시간/공간 복잡도를 명시적으로 분석한다
- 엣지 케이스를 체계적으로 열거한다 (빈 입력, 단일 요소, 최대값, 음수)
- 불변식(invariant)을 코드 내 주석으로 기록한다
- 성능 크리티컬 경로는 벤치마크 테스트 포함

## 주의 사항
- 조기 최적화 금지 — 먼저 정확성, 그 다음 성능
- 재귀는 스택 오버플로 위험 검토 (깊이 제한 또는 반복 변환)
- 부동소수점 비교에 epsilon 사용
- 동시성/병렬 처리 시 race condition 검토

## 검증 포인트
- 모든 엣지 케이스에 대한 테스트가 있는가?
- 시간/공간 복잡도가 요구사항을 충족하는가?
- 큰 입력에서 타임아웃 없이 동작하는가?
- 수학적 정확성이 검증되는가?
