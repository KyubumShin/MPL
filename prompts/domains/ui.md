# Domain: UI (프론트엔드)

## 핵심 원칙
- 컴포넌트는 단일 책임 원칙 준수
- 상태 관리는 최소 범위에서 (local → context → global)
- 접근성(a11y) 기본 준수 (aria, semantic HTML, keyboard nav)
- 반응형 디자인 고려 (mobile-first 또는 프로젝트 컨벤션)

## 주의 사항
- 인라인 스타일 남용 금지 — 프로젝트 스타일 시스템 사용
- 불필요한 리렌더링 방지 (memo, useMemo, useCallback 적절히)
- 하드코딩된 문자열 금지 — i18n 또는 상수 파일 사용
- 이미지/미디어는 최적화된 형식과 lazy loading

## 검증 포인트
- 컴포넌트가 다양한 뷰포트에서 올바르게 렌더링되는가?
- 키보드만으로 모든 기능에 접근 가능한가?
- 에러 상태와 로딩 상태가 처리되는가?
- 기존 디자인 시스템/토큰과 일치하는가?
