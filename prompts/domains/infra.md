# Domain: Infra (인프라/설정)

## 핵심 원칙
- 환경별 설정은 환경 변수로 관리 (하드코딩 금지)
- 비밀 정보는 절대 코드에 포함하지 않는다
- 설정 변경은 롤백 가능해야 한다
- CI/CD 파이프라인 변경은 dry-run 테스트 필수

## 주의 사항
- .env 파일을 git에 커밋하지 않는다
- Docker 이미지는 최소 크기 (multi-stage build)
- 포트 충돌, 볼륨 마운트 경로 검증
- 의존성 버전 고정 (lock 파일 존재 확인)

## 검증 포인트
- 모든 환경 변수가 문서화되어 있는가?
- Docker build/run이 에러 없이 성공하는가?
- CI 파이프라인이 정상 동작하는가?
- 비밀 정보가 로그에 노출되지 않는가?
