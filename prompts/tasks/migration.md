# Task Type: Migration (마이그레이션/전환)

## 핵심 원칙
- 롤백 전략을 사전에 수립하고 문서화한 뒤 작업 시작
- 하위 호환성 유지: 구/신 시스템이 공존하는 전환 기간을 명시
- 데이터 마이그레이션과 코드 마이그레이션은 별도 단계로 분리
- 점진적 전환: 전체를 한 번에 교체하지 않고 단계적으로 이행
- 각 단계의 성공 기준을 정의한 뒤 다음 단계로 진행

## 주의 사항
- 구 시스템 제거는 신 시스템 안정성 확인 후 별도 작업으로 수행
- 데이터 손실 가능성이 있는 변환은 드라이런(dry-run) 먼저 실행
- 외부 시스템(클라이언트, 파트너 API) 의존성 목록 사전 파악
- 구/신 공존 기간 중 이중 쓰기 또는 포워딩 전략 명시

## 검증 포인트
- 롤백 절차가 실제로 동작하는가?
- 기존 기능이 마이그레이션 후에도 정상 동작하는가?
- 새 시스템의 핵심 기능이 검증되었는가?
- 데이터 정합성이 유지되는가?
