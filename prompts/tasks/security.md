# Task Type: Security (보안 강화)

## 핵심 원칙
- OWASP Top 10 체크리스트를 기준으로 적용 범위 결정
- 모든 외부 입력은 sanitize + validate (둘 다 필수)
- 인증(Authentication)과 인가(Authorization)를 별도 레이어로 분리
- 최소 권한 원칙: 필요한 범위만 접근 허용
- 비밀 정보(키, 패스워드, 토큰)는 소스 코드·로그에서 완전 제거

## 주의 사항
- 보안 수정이 기능 동작에 미치는 영향을 반드시 검토
- 암호화 알고리즘은 직접 구현하지 않고 검증된 라이브러리 사용
- 에러 메시지에 내부 구현 세부사항 노출 금지
- 보안 패치는 별도 커밋으로 분리 (변경 이력 추적 용이)

## 검증 포인트
- 알려진 공격 벡터(XSS, SQLi, CSRF 등)가 차단되는가?
- 인증 없이 보호된 리소스에 접근이 불가능한가?
- 민감 정보가 응답·로그·에러 메시지에 노출되지 않는가?
- 입력 검증이 서버 측에서도 수행되는가 (클라이언트 측만 아닌가)?
